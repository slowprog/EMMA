Getting Started

A quick & easy start to creating responsive HTML emails.

The Boilerplate

In the downloaded files you'll find a boilerplate.html file with the pre-setup code below to get you started quickly.

You can also just grab the code below.

Make sure to take note of the stylesheet workflow in Emma. The Emma framework CSS file is linked into the boilerplate through the <link rel="stylesheet" href="assets/css/emma.css" /> statement.

When you are done with your development, you have to paste the Emma CSS into the <head> area using a <style> tag. Any custom CSS styles should also be included like this.

Make sure not to include the media queries before running all of this code through an Inliner tool. After inlining all the CSS, paste the media queries back into the <head> area.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
  <html xmlns="http://www.w3.org/1999/xhtml">
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
    <meta name="viewport" content="width=device-width"/>

    <link rel="stylesheet" href="emma.css"> <!-- For Development Only -->

    <style type="text/css">
        /* Paste Emma framework CSS here before running Inliner */
    </style>
    <style type="text/css">
        /* Your custom CSS goes here */
    </style>
  </head>
  <body>
    <table class="body">
        <tr>
            <td class="center" align="center" valign="top">
            <center>
                <table class="container">
                    <tr>
                        <td>
                            <table class="row">
                                <tr>
                                    <td class="wrapper last">
                                        <table class="twelve columns">
                                            <tr>
                                                <td class="panel">
                                                .12col starter row
                                                </td>
                                                <td class="expander"></td>
                                            </tr>
                                        </table>
                                    </td>
                                </tr>
                            </table>
                        </td>
                    </tr>
                </table>
            </center>
            </td>
        </tr>
    </table>
  </body>
  </html>

Theming Emma

Emma uses LESS to easily style all the components.

LESS CSS

When you open up the variables.less file in Emma's assets folder, you'll find a range of style variables that you can change as you want. Although the style variables are very intuitive to change, you will need some knowledge of the LESS CSS language or at least know how to compile the LESS files for your changes to take effect.

Emma
    assets
        css
        less
            variables.less

Style Variables

You can use the default styles as is to quickly mockup and layout your email to get a feel for the content and flow of your email. Afterwards, easily change the style variables to make Emma look the way you like, match it to your design or brand.

/* Type Setup */
@sansFontFamily:        "Helvetica Neue", Helvetica, Arial, sans-serif;
@serifFontFamily:       Rockwell, "Courier Bold", Courier, Georgia, Times, serif;
@monoFontFamily:        "Courier New", Courier, "Lucida Sans Typewriter", monospace;
/* Base Typography */
@baseFontFamily:        @sansFontFamily;
@baseFontSize:          14px;
@baseFontWeight:        normal;
@baseFontColor:         @black;
@baseLineHeight:        floor(@baseFontSize * 1.45); //20px

Don't know LESS?

If you don't know how to work with LESS, you can just start using Emma as is, the basic styles are meant to give you a great looking email by default.

Grid

Powerful responsive layouts that simply work.

Structure

Emma is based on a 12-column grid, similar to frameworks like Bootstrap, but it has a fixed-width container of 580px (universally recommended display widths for email is 550 - 600px).

Emma's grid is made up of 4 main components:

Containers

Keeps the layout wrapped to a fixed 580px width on large displays and 90% fluid width on small displays.

Rows

Creates vertical whitespace between content blocks.

Wrappers

Creates and maintains the gutters between columns.

Columns

Creates the width of each column in the 12-column grid, and contains the content.

The 4 grid components in code

<table class="container">
    <tr>
        <td>
            <table class="row">
                <tr>
                    <td class="wrapper last">
                        <table class="twelve columns">
                            <tr>
                                <td>
                                content in a row, 12 columns wide.
                                </td>
                                <td class="expander"></td>
                            </tr>
                        </table>
                    </td>
                </tr>
            </table>
        </td>
    </tr>
</table>

Breakdown

These are all the main grid elements and what they do.

Element Class Description
table container Constrains content to a width of 580px and centers it. On smaller displays it becomes fluid and 90% of the display width.
table row Separates each row of content with white space.
td wrapper Wraps each .columns table, creates a gutter between columns and forces columns to expand to full width on small displays.
td last Applied to the last .wrapper td in each row for the gutters to work properly. Even if you have only one .wrapper td in a row.
table {number} Numbered columns, from one to twelve (spelled out). Creates the width for your .columns tables. Each row must contain twelve columns, including offset columns.
table columns Columns are n-twelfths of the width of the 580px .container on large displays, and expand to the full-width of the .container on small displays.
td expander Added after the content td within a .columns table. It's an empty and invisible element, but it allows the content td to expand to the full width of the .container on small displays.

Sub-Grids

For even more control over your layouts, use nested grids.

Nesting Grids

Emma gives you the ability to finely control your layouts through nestable grids called sub-grids.

By adding a .sub-columns class and a numbered class (exactly like normal grids) to a content td inside the main .columns <table> you can easily sub-divide the main columned row.

Just like the main grid, you have to add a .last class to the last .sub-columns td in the row.

<table class="container">
    <tr>
        <td>

            <table class="row">
                <tr>
                    <td class="wrapper">

                        <table class="four columns">

Block-Grid

Evenly sized blocks that are media query independent.

Evenly Spaced Content

Emma's block grid is an alternative way to layout elements when the normal grid won't do.

Block-Grid elements align left by default and push each other down to the next row as the display size gets smaller, without making use of media queries to do so.

Block-grids work slightly different from the normal grid, where you once had a .row, you now replace the .row <table> with a .block-grid and a numbered class to call the amount of evenly size blocks you need.

Make sure your <td> ending and beginning tags are stacked next to each other without spaces. Some clients render these spaces and break the grid.

<table class="container">
    <tr>
        <td>

            <table class="block-grid two-up">
                <tr>
                    <td>
                        two-up
                    </td><td> <!-- Watch out for spacing between the td tags.
                                   Some clients render these spaces. -->
                        two-up
                    </td>
                </tr>
            </table>

            <table class="block-grid four-up">
                <tr>
                    <td>
                        four-up
                    </td><td> <!-- no spacing between tags -->
                        four-up
                    </td><td> <!-- no spacing between tags -->
                        four-up
                    </td><td> <!-- no spacing between tags -->
                        four-up
                    </td>
                </tr>
            </table>

        </td>
    </tr>
</table>

Block-Grids Example

Making Block-Grids Fluid

In some cases you want to use Block-Grids for content but still make them flow the same way normal grids would when it comes to mobile displays.

In this instance you want to enable a media query to do just that.

@media only screen and (max-width: 600px) {
    table[class="container"] .block-grid td {
        width: 95% !important; /* 95% to mimic the padding of the normal grid. */
    }
}

Alignment Helpers

Centering Content

To center content you need to apply special .center classes to the content you want to center.

For text, apply a .center to the <td> that contains the text.

For Images, you need to also apply a second .center to the actual <img>.

And for maximum support in Outlook.com, we recommend you place the <center> tag around the actual content you wish to center.

Text Padding

When you have columns of text, often the spacing between the columns of text become an issue. Often all you want is just more white space between text columns. In this event, use the .text-pad, .left-text-pad and .right-text-pad classes as you need them.

The .text-pad class gives you 10px of white space on both sides of content. Even though the class is named as "text padding" it works equally well on other content like images and buttons, making sure that your content doesn't flow right up against the edges of your containers.

Apply the .text-pad class to the <td> of a .columns <table>.

.left-text-pad and .right-text-pad does exactly what you would expect, but note that the left only padding and right only padding applies to large displays only, when the display gets small, padding gets added to both left and right sides.

Text Padding Example

Text Padding Code Example

<table class="row">
    <tr>
        <td class="wrapper last offset-by-three">

            <table class="six columns">
                <tr>
                    <td class="text-pad">
                        <p>Using .text-pad content is padded on both left and right sides.</p>
                    </td>
                    <td class="expander"></td>
                </tr>
            </table>

        </td>
    </tr>
</table>

Column Expanders

Emma requires these expander <td>'s to make the grid work correctly on mobile displays.

On small displays or mobile displays, the .columns tables go full-width or 100% of the container and stacks vertically. Some email clients don't expand these columns properly, so the extra expander td is necessary to force the columns to expand to 100% or full-width of its container.

<table class="row">
    <tr>
        <td class="wrapper last">

            <table class="twelve columns">
                <tr>
                    <td>
                        <p>a small amount of content.</p> <!-- Not enough content to push container
                                                               open all the way on small displays -->
                    </td>
                    <td class="expander"></td> <!-- Use expander to fix this problem
                                                    and push containers full-width on small displays -->
                </tr>
            </table>

        </td>
    </tr>
</table>

Visibility Helpers

Selectively show and hide content

You can add a level of responsivity to your content by hiding or showing content on either large or small displays, by using .show-for-small or .hide-for-small.

Visibility classes can be used on any element. Outlook lacks support for visibility classes, but using them in conjunction with conditional comments ensures that the content is properly hidden or shown in Outlook 2007/2010/2013.

Note: When using a visibility class on an image, make sure to apply it to the parent element, not the image itself.

Visibility Class Code Example

<table class="row">
    <tr>
        <td class="wrapper last">

            <!--[if !mso]><!-- -->
            <table class="twelve columns show-for-small">  <!-- Only gets shown on small displays -->
                <tr>
                    <td>
                        .show-for-small
                    </td>
                    <td class="expander"></td>
                </tr>
            </table>
            <!--<![endif]-->

            <table class="twelve columns hide-for-small">  <!-- Only gets shown on large displays -->
                <tr>
                    <td>
                        .hide-for-small
                    </td>
                    <td class="expander"></td>
                </tr>
            </table>

        </td>
    </tr>
</table>

Images

Add awesome images (even Retina versions) to your emails.

Correct Image Markup

Emma loves Retina images and makes them simple to use. To use high resolution images in your email, make sure to assign a height and width attribute on the <img> tag. Be sure that your width attribute corresponds to one of the column sizes as indicated below.

On large displays the images will be displayed at the width of the .columns table and on smaller displays the images will expand to 100% width of it's parent element.

Image Code Markup Example

<table class="row">
    <tr>
        <td class="wrapper">

            <table class="three columns">
                <tr>
                    <td>

                        <img height="130" width="130" src="image-url"> <!-- Width attribute matching columns table -->

                    </td>
                    <td class="expander"></td>
                </tr>
            </table>

        </td>
    </tr>
</table>

Sizing Images

For normal resolution images, use any of the widths below to physically size your images. These images will display at exactly these sizes up to 600px wide.

For Retina Images, use whatever resolution suits your needs. Any image sized larger than 600px wide will be treated as Retina. These images will be restricted in width by the .columns table for larger displays and automatically fallback to their high resolution physical size and display 100% width of their parent element on small displays.

Image height should always be set in relation to the width and carefully considered.

Supported Email Clients

We thoroughly test all major email clients for desktop and mobile.

Outlook First Approach

Emma takes a progressive enhancement approach to email design. Historically, Outlook has always been the 3 legged dog from hell when it comes to html email, so it makes sense to appease Outlook first before moving on to more advanced email clients. But Emma does include styles like rounded corners and background images that will look great in email clients that support them.

Testing Clients

We do our best to test as thoroughly as we can, and we use a specialised email testing service namely Email On Acid to do so. Email On Acid is a world leader in email client testing, and through them we test over 70 different email client, web browser and operating system combinations, from desktop through to mobile.

While our testing is extensive, the list of supported email clients below have been chosen specifically for their market share, popularity and reliability.
